
# 一、合并区间

## 题型
- 排序
- 区间
- 贪心
- 模拟

## 核心思路

这题的核心是：**先排序，再合并。**

因为区间如果是乱序的，你无法高效判断哪些区间可以合并。  
所以第一步先**按区间左端点**从小到大排序。

排序后，遍历每个区间，并维护一个结果数组 `res`：

- 如果 `res` 为空，或者当前区间和 `res` 最后一个区间**不重叠**，就直接加入 `res`
- 如果当前区间和 `res` 最后一个区间**重叠**，就把最后一个区间的右端点更新为两者右端点的最大值

这样遍历一遍后，`res` 中就是所有合并后的区间。

---

## 为什么想到这个方法

因为这题属于非常典型的 **区间问题**。

区间题里常见的第一反应就是：

- 先排序
- 再看是否能与前面的区间合并

这题排序后有一个很重要的性质：

**当前区间只需要和结果数组最后一个区间比较。**

因为一旦按左端点升序排列：

- 如果当前区间和最后一个区间都不重叠
- 那它更不可能和更前面的区间重叠

所以不需要回头比较所有区间。

这就把原本可能复杂的比较过程，变成了线性扫描。

## 模板代码


```python
from typing import List

class Solution:
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        if not intervals:
            return []

        intervals.sort(key=lambda x: x[0])
        res = []

        for interval in intervals:
            if not res or res[-1][1] < interval[0]:
                res.append(interval)
            else:
                res[-1][1] = max(res[-1][1], interval[1])

        return res
````

## 面试官会问的点

### 1. 为什么先排序？

因为排序后区间按左端点有序，当前区间只需要和结果数组最后一个区间比较，不需要回头看前面所有区间。

---

### 2. 为什么只看最后一个区间？

因为已经按左端点升序排序。当前区间如果和最后一个区间都不重叠，那么它也一定不会和更前面的区间重叠。

---

### 3. 如何判断是否重叠？

在排序前提下，如果：

```python
res[-1][1] < interval[0]
```

则不重叠，否则重叠。

---

### 4. 为什么只更新右端点？

因为排序后前一个区间左端点一定不大于当前区间左端点，所以左端点不用改，只更新右端点为最大值即可。

---

### 5. `lambda` 是什么？

`lambda` 是匿名函数，也就是临时函数。这里用于指定排序规则：按区间左端点排序。

---

### 6. 这题是什么类型？

这是典型的 **区间合并题**，属于 **排序 + 贪心 / 模拟**。

---

## 易错点

### 1. 忘记先排序

不排序的话，后面的合并逻辑不成立。

---

### 2. 把“不重叠”的条件写错

正确写法是：

```python
res[-1][1] < interval[0]
```

不是比较左端点，也不是比较整个区间。

---

### 3. 合并时把左端点也改了

左端点不需要改，因为排序后前面的左端点一定更小或相等。

---

### 4. 没有理解 `res[-1]`

`res[-1]` 表示结果数组最后一个区间，不是原数组最后一个区间。

---

## 复杂度分析

* 时间复杂度：`O(n log n)`

  * 排序：`O(n log n)`
  * 遍历合并：`O(n)`
  * 总复杂度由排序主导

* 空间复杂度：`O(n)`

  * 最坏情况下所有区间都不重叠，结果数组需要存下所有区间
