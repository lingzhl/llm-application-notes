
# 一、岛屿数量

- 首次做题日期：2026.4.21
- 复刷日期：

这题本质上是在问：

**二维网格中有多少个连通块。**

这里：

* `'1'` 表示陆地
* `'0'` 表示水
* 一个岛屿 = 一片上下左右连在一起的陆地

所以题目其实就是：

**数一共有多少个由 `'1'` 组成的连通区域。**

---

## 快速记忆思路

## 核心一句话

**遇到一个新的 `'1'`，岛屿数量加一，然后把和它连着的整块陆地全部淹掉。**

这就是这题最核心的记忆方式。

---

## 为什么这样做

当你在遍历网格时：

* 如果当前是 `'0'`，说明是水，跳过
* 如果当前是 `'1'`，说明发现了一座新岛屿

这时候：

1. 岛屿数量 `count += 1`
2. 从这个点出发，用 DFS 或 BFS 把整座岛屿全部标记成 `'0'`

这样后面再遍历到这块区域时，就不会重复计数了。

---

## 最短口诀

**见 1 计数，DFS 淹岛。**

或者：

**找到一块陆地，就把整座岛沉掉。**

---

遇到这种题就要想到：

* DFS
* BFS
* 并查集（更进阶）

---

## 3. DFS 的核心作用

DFS 在这里不是找最短路，而是：

**从一个 `'1'` 出发，把整座岛全部访问完。**

也就是把这个连通区域完整扫一遍。

---

## 4. 原地修改

这题用了：

```python

```

意思是：

**访问过的陆地直接改成水**

这样就不需要额外的 `visited` 数组了。

---

## 5. 四个方向

这题只考虑：

* 上
* 下
* 左
* 右

不算对角线。

## 代码逐行解释

先看代码：

```python
class Solution:
    def numIslands(self, grid):
        if not grid or not grid[0]:
            return 0

        rows, cols = len(grid), len(grid[0])
        count = 0

        def dfs(r, c):
            # 边界检查 & 避免重复访问
            if r < 0 or c < 0 or r >= rows or c >= cols or grid[r][c] == '0':
                return
            # 标记访问过的陆地
            grid[r][c] = '0'
            # 递归访问上下左右
            dfs(r + 1, c)
            dfs(r - 1, c)
            dfs(r, c + 1)
            dfs(r, c - 1)

        for r in range(rows):
            for c in range(cols):
                if grid[r][c] == '1':
                    count += 1
                    dfs(r, c)

        return count
```

## 时间复杂度和空间复杂度

## 时间复杂度：`O(n * m)`

为什么？

* 网格一共有 `n * m` 个格子
* 每个格子最多被访问一次
* 所以总复杂度是：

```python
O(n * m)
```

---

## 空间复杂度：`O(n * m)`（最坏情况）

如果整张图全是陆地：

* DFS 递归深度可能达到 `n * m`

所以最坏情况下递归栈空间是：

```python
O(n * m)
```

---

## 如果面试官追问“额外空间是不是 O(1)”

你可以这样答：

> 如果不算递归调用栈，这里是原地修改网格，没有额外 visited 数组，所以额外显式空间接近 O(1)。但严格算递归栈的话，最坏情况是 O(n * m)。

这个回答比较完整。

## 面试官会问的点

## 1. 为什么遇到 `'1'` 就可以 count + 1？

答：

> 因为当前遍历到的 `'1'` 还没有被之前的 DFS 淹掉，所以它一定是一座新岛屿的起点。

---

## 2. 为什么 DFS 后要把整座岛标记成 `'0'`？

答：

> 为了避免重复计数。如果不标记，后面遍历到这座岛上的其他格子时，会重复把它当成新岛屿。

---

## 3. 为什么这题可以用 DFS/BFS？

答：

> 因为这题本质是求二维网格中的连通块个数。DFS 和 BFS 都能完整遍历一个连通区域。

---

## 4. DFS 和 BFS 有什么区别？

答：

> 都能做。DFS 代码更简洁；BFS 不会有递归栈溢出问题，更适合更大的网格。

---

## 5. 为什么时间复杂度是 O(n*m)？

答：

> 因为每个格子最多被访问一次，不会重复搜索。

---

## 6. 如果不想修改原数组怎么办？

答：

> 可以额外开一个 `visited` 数组来记录访问状态。

# 二、全排列

## 知识点

## 1. 回溯法

这题最核心的知识点就是：

**回溯（Backtracking）**

回溯适合这种题：

* 全排列
* 组合
* 子集
* N 皇后
* 路径搜索

它的本质是：

**做选择 → 递归 → 撤销选择**

---

## 2. 全排列

全排列的意思是：

**把数组中所有元素按不同顺序全部排出来**

例如：

```python id="43714"

```

```python id="43715"

```

## 3. used 数组

```python id="43716"

```

它的作用是：

**记录某个数字当前是否已经被放进 current 路径里**

因为排列里每个数字只能用一次，所以要标记。

---

## 4. current 路径

```python id="43717"

```

表示：

例如递归中可能有：

```python id="43718"

```

---

## 5. result 结果数组

```python id="43719"

```

---

## 核心思路一句话

**从空路径开始，每次选一个没用过的数加入路径，直到路径长度等于数组长度。**

---

## 详细思路

1. 用 `current` 保存当前排列
2. 用 `used` 记录每个数字是否已经用过
3. 每次递归时，遍历数组中的每个数字：

   * 如果已经用过，跳过
   * 如果没用过，加入 `current`
4. 递归进入下一层
5. 递归回来后，撤销刚才的选择
6. 如果 `current` 长度等于 `nums` 长度，说明得到一个完整排列，加入答案

---

## 最短记忆口诀

**路径 current，标记 used；选一个，递归；回来撤销。**

---

## 关键符号含义

先看代码：

```python id="43720"
class Solution:
    def permute(self, nums):
        result = []
        used = [False] * len(nums)
        current = []

        def backtrack():
            if len(current) == len(nums):
                result.append(current[:])
                return
            for i in range(len(nums)):
                if used[i]:
                    continue
                used[i] = True
                current.append(nums[i])
                backtrack()
                current.pop()
                used[i] = False

        backtrack()
        return result
```

## 时间复杂度

### 为什么

* 一共有 `n!` 种排列
* 每个排列长度是 `n`
* 加入答案时需要拷贝 `current[:]`，是 `O(n)`

所以总时间复杂度：

```python id="43740"
O(n * n!)
```

---

## 空间复杂度

**O(n)**（不算结果数组）

### 为什么

* 递归深度最多 `n`
* `used` 长度是 `n`
* `current` 长度也是 `n`

所以辅助空间是：

```python id="43741"
O(n)
```

如果把最终结果 `result` 也算进去，总存储空间会更大，为：

```python id="43742"
O(n * n!)
```

---

### 面试官会问的知识点

## 1. 为什么用回溯？

你可以答：

> 因为这题要求枚举所有可能排列，本质是一个穷举搜索问题，回溯可以系统地尝试每一种选择。

---

## 2. used 数组是干什么的？

你可以答：

> used 用来标记当前数字是否已经加入当前排列，避免同一个数字在一个排列里重复使用。

---

## 3. 为什么要 `current[:]`？

你可以答：

> 因为 current 是同一个列表对象，后续回溯过程中会不断修改，所以加入结果时必须拷贝一份当前状态。

---

## 4. 为什么要回溯撤销？

你可以答：

> 因为当前选择只在当前分支有效，递归回来后必须恢复现场，才能继续尝试别的选择。

---

## 5. 这题和组合题的区别是什么？

你可以答：

> 排列要考虑顺序，组合不考虑顺序。例如 `[1,2]` 和 `[2,1]` 在排列里不同，在组合里相同。

---

## 6. 回溯模板是什么？

你可以答：

> 回溯模板通常是：终止条件、枚举选择、做选择、递归、撤销选择。

# 六、最适合背的总结

## 核心思路

* 用 `current` 记录路径
* 用 `used` 记录是否已使用
* 每次选一个没用过的数字加入路径
* 路径长度够了就加入结果
* 回来后撤销选择

## 一句话记忆

**路径 current，标记 used；选一个，递归；回来撤销。**

## 复杂度

* 时间：`O(n * n!)`
* 空间：`O(n)`

# 三、分别是什么

---

## 1️⃣ DFS 是什么

👉 **搜索方式（策略）**

特点：

👉 一条路走到底，再回头

---

### 例子

```text
走迷宫：
一直往前走 → 撞墙 → 回头 → 换路
```

---

## 2️⃣ 递归是什么

👉 **写代码的方法**

特点：

👉 函数自己调用自己

---

### 例子

```python
def f(n):
    if n == 0:
        return
    f(n-1)
```

👉 只是“调用方式”，没有具体“问题含义”

---

## 3️⃣ 回溯是什么

👉 **解决问题的算法思想**

特点：

👉 尝试所有可能 + 失败就撤销

---

### 例子（全排列）

```text
选一个数 → 继续选 → 选满 → 记录
          ↓
        回来撤销 → 换一个数
```
