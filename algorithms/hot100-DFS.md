
# 一、岛屿数量

- 首次做题日期：2026.4.21
- 复刷日期：

这题本质上是在问：

**二维网格中有多少个连通块。**

这里：

* `'1'` 表示陆地
* `'0'` 表示水
* 一个岛屿 = 一片上下左右连在一起的陆地

所以题目其实就是：

**数一共有多少个由 `'1'` 组成的连通区域。**

---

## 快速记忆思路

## 核心一句话

**遇到一个新的 `'1'`，岛屿数量加一，然后把和它连着的整块陆地全部淹掉。**

这就是这题最核心的记忆方式。

---

## 为什么这样做

当你在遍历网格时：

* 如果当前是 `'0'`，说明是水，跳过
* 如果当前是 `'1'`，说明发现了一座新岛屿

这时候：

1. 岛屿数量 `count += 1`
2. 从这个点出发，用 DFS 或 BFS 把整座岛屿全部标记成 `'0'`

这样后面再遍历到这块区域时，就不会重复计数了。

---

## 最短口诀

**见 1 计数，DFS 淹岛。**

或者：

**找到一块陆地，就把整座岛沉掉。**

---

DFS 在这里不是找最短路，而是：

## 代码逐行解释

先看代码：

```python
class Solution:
    def numIslands(self, grid):
        if not grid or not grid[0]:
            return 0

        rows, cols = len(grid), len(grid[0])
        count = 0

        def dfs(r, c):
            # 边界检查 & 避免重复访问
            if r < 0 or c < 0 or r >= rows or c >= cols or grid[r][c] == '0':
                return
            # 标记访问过的陆地
            grid[r][c] = '0'
            # 递归访问上下左右
            dfs(r + 1, c)
            dfs(r - 1, c)
            dfs(r, c + 1)
            dfs(r, c - 1)

        for r in range(rows):
            for c in range(cols):
                if grid[r][c] == '1':
                    count += 1
                    dfs(r, c)

        return count
```

* `r < 0`：跑到最上面外面去了
* `c < 0`：跑到最左边外面去了
* `r >= rows`：跑到最下面外面去了
* `c >= cols`：跑到最右边外面去了
* `grid[r][c] == '0'`：当前是水，或者已经访问过

## 时间复杂度和空间复杂度

## 时间复杂度：`O(n * m)`

为什么？

* 网格一共有 `n * m` 个格子
* 每个格子最多被访问一次
* 所以总复杂度是：

```python
O(n * m)
```

---

## 空间复杂度：`O(n * m)`（最坏情况）

如果整张图全是陆地：

* DFS 递归深度可能达到 `n * m`

所以最坏情况下递归栈空间是：

```python
O(n * m)
```

---

## 如果面试官追问“额外空间是不是 O(1)”

你可以这样答：

> 如果不算递归调用栈，这里是原地修改网格，没有额外 visited 数组，所以额外显式空间接近 O(1)。但严格算递归栈的话，最坏情况是 O(n * m)。

这个回答比较完整。

## 面试官会问的点

## 1. 为什么遇到 `'1'` 就可以 count + 1？

答：

> 因为当前遍历到的 `'1'` 还没有被之前的 DFS 淹掉，所以它一定是一座新岛屿的起点。

---

## 2. 为什么 DFS 后要把整座岛标记成 `'0'`？

答：

> 为了避免重复计数。如果不标记，后面遍历到这座岛上的其他格子时，会重复把它当成新岛屿。

---

## 3. 为什么这题可以用 DFS/BFS？

答：

> 因为这题本质是求二维网格中的连通块个数。DFS 和 BFS 都能完整遍历一个连通区域。

---

## 4. DFS 和 BFS 有什么区别？

答：

> 都能做。DFS 代码更简洁；BFS 不会有递归栈溢出问题，更适合更大的网格。

---

## 5. 为什么时间复杂度是 O(n*m)？

答：

> 因为每个格子最多被访问一次，不会重复搜索。

---

## 6. 如果不想修改原数组怎么办？

答：

> 可以额外开一个 `visited` 数组来记录访问状态。

---

## 标准解题思路

你面试时可以这样说：

> 这题本质上是统计二维网格中连通区域的个数。遍历整个网格时，遇到一个 `'1'`，说明发现了一座新的岛屿，于是岛屿数量加一，并从该位置出发做 DFS，把与它上下左右相连的所有陆地都标记成 `'0'`，避免后续重复计数。这样每个格子最多被访问一次，所以时间复杂度是 `O(n*m)`。
