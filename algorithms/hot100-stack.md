# 一、有效的括号
- 首次做题日期：2026-04-17
- 复刷日期：2026-04-18

## 题型
- 栈
- 哈希表
- 模拟

## 核心思路

- 这题的核心是用 **栈** 来维护“还没有匹配的左括号”。

- 遍历字符串时：

- 遇到左括号，就先入栈
- 遇到右括号，就拿它和栈顶的左括号进行匹配
- 如果栈为空，或者栈顶括号和当前右括号不匹配，说明字符串不合法，直接返回 `False`

最后如果所有字符遍历完后，栈刚好为空，说明所有括号都成功一一匹配，返回 `True`。

---

## 为什么想到这个方法

因为括号匹配有一个很明显的特点：

**后出现的左括号，要先被匹配。**

例如：

```python
([{}])

匹配顺序是：

{}
[]
()

也就是说，最后进去的左括号，最先出来匹配，这正好符合栈的“后进先出”特性。

所以一看到这种：

括号匹配
嵌套结构
最近匹配

就要优先想到 栈。
```

## 模板代码
```python

def isValid(s):
    stack = []
    mp = {')': '(', ']': '[', '}': '{'}

    for ch in s:
        if ch in '([{':
            stack.append(ch)
        elif ch in ')]}':
            if not stack or stack.pop() != mp[ch]:
                return False

    return not stack
```

## 易错点

#### 1.忘记先判断栈空
```python

遇到右括号时，不能直接 pop()，因为如果栈为空会报错。

错误写法：

if stack.pop() != mp[ch]:

正确写法：

if not stack or stack.pop() != mp[ch]:
```


#### 2. 遍历结束后忘记判断栈是否为空
```python
比如：

"((("

遍历过程中不会报错，但显然不是合法括号串。

所以最后一定要写：

return len(stack) == 0
```

#### 3. 字典映射方向写反
```python
推荐写成：

mp = {')': '(', ']': '[', '}': '{'}

因为当前遍历到的是 右括号，要快速找到它应该对应哪个左括号。
```

#### 4. 把 Python 的 False 写成 false
```python
Python 里布尔值首字母必须大写：

False
True
```

#### 5. 只想到统计数量，没有考虑顺序
```
例如：

")("

左右括号数量一样，但顺序错了，依然不合法。

所以这题不能只数个数，必须考虑：

顺序
类型
嵌套关系
```

## 复杂度分析
- 时间复杂度：O(n)
只遍历字符串一次，每个字符最多入栈或出栈一次。
- 空间复杂度：O(n)
最坏情况下所有字符都是左括号，栈需要存下所有字符。

## 面试回答模板

- 这题我会用栈来做。因为括号匹配具有明显的后进先出特征，后出现的左括号需要先和右括号匹配。
- 遍历字符串时，遇到左括号就入栈；遇到右括号就检查栈是否为空，并判断它是否和栈顶左括号匹配。
- 如果中途出现栈空或者匹配失败，就直接返回 False。遍历结束后，如果栈为空，说明所有括号都成功匹配，返回 True。
- 时间复杂度是 O(n)，空间复杂度是 O(n)。

## 一句话记忆

左括号入栈，右括号和栈顶匹配，不合法立即返回，最后栈空才算成功。

# 二、 每日温度
````markdown
- 首次做题日期：2026-4-18
- 复刷日期：

## 题型
- 单调栈
- 栈
- 数组

## 核心思路

这题的本质是：**找每个位置右边第一个比它大的元素。**

对于第 `i` 天温度，如果后面某一天温度更高，就返回要等几天；如果后面没有更高温度，就返回 `0`。

这类“右边第一个更大元素”问题，经典做法就是 **单调栈**。

具体做法是：

- 维护一个**单调递减栈**
- 栈里存的是**下标**
- 这些下标对应的温度值，从栈底到栈顶递减

遍历数组时：

- 如果当前温度不高于栈顶对应温度，就直接入栈
- 如果当前温度更高，说明当前这一天就是栈顶那一天右边第一个更高温度
- 这时不断弹栈，并计算等待天数

最后栈里剩下的元素，说明它们右边没有更高温度，答案保持 `0` 即可。

---

## 为什么想到这个方法

因为这题不是让你找“最大值”，而是找：

**右边第一个更大值。**

只要题目出现下面这种特征，就要想到单调栈：

- 右边第一个更大元素
- 右边第一个更小元素
- 找最近满足条件的位置
- 每个元素都要快速找到它后面第一个符合条件的位置

这题中：

- 当前温度如果更高
- 就能帮前面那些更低温度的位置“结算答案”

所以适合用栈来维护那些“还没找到更高温度的位置”。

---

## 模板代码

```python
class Solution:
    def dailyTemperatures(self, temperatures: list) -> list:
        n = len(temperatures)
        answer = [0] * n
        stack = []

        for i in range(n):
            while stack and temperatures[i] > temperatures[stack[-1]]:
                idx = stack.pop()
                answer[idx] = i - idx
            stack.append(i)

        return answer
````

---

## 要背的知识点

### 1. 单调栈

这题的核心知识点就是 **单调栈**。

这里维护的是：

**单调递减栈**

注意：

* 栈里存的是下标
* 下标对应的温度值递减

---

### 2. 右边第一个更大元素

这题本质上就是经典问题：

**Next Greater Element（下一个更大元素）**

区别只是：

* 普通题返回更大的值
* 这题返回相隔几天

---

### 3. 为什么栈里存下标

因为题目要求返回：

```python
i - idx
```

也就是相差几天。

如果只存温度，不存下标，就没法计算距离。

---

### 4. while 弹栈结算

当前温度更高时，说明它可以给前面较低温度的位置结算答案：

* 弹栈
* 取出旧下标
* 计算 `i - idx`

这是单调栈题最典型的套路。

---

## 面试官会问的点

### 1. 为什么想到单调栈？

因为这题要求的是每个位置右边第一个更大元素，这类题适合用单调栈在线性时间内解决。

---

### 2. 为什么是单调递减栈？

因为我们要找“更大”的元素。

如果当前元素比栈顶更大，就说明它是栈顶位置的答案，所以栈里保留的是一串还没找到更大值的位置，它们对应的值保持递减更方便处理。

---

### 3. 为什么存下标，不存温度？

因为最后要返回间隔天数：

```python
i - idx
```

必须知道是哪一天，不能只知道温度值。

---

### 4. 为什么时间复杂度是 `O(n)`，while 不会退化成 `O(n^2)` 吗？

不会。

因为每个下标最多：

* 入栈一次
* 出栈一次

所以总操作次数不会超过 `2n`，整体复杂度仍然是 `O(n)`。

---

### 5. 如果后面没有更高温度怎么办？

因为答案数组初始化为全 `0`：

```python
answer = [0] * n
```

那些没有被更新的位置，自然就是 `0`。

---

### 6. 这题和“下一个更大元素”是什么关系？

这题本质上就是“下一个更大元素”的变形题。

只是返回值不是“更大的元素”，而是“到那个更大元素的距离”。

---

### 7. 如果改成找右边第一个更小元素怎么办？

把单调性反过来即可。

例如维护单调递增栈。

---

## 易错点

### 1. 栈里存成温度而不是下标

错误原因：

* 没法计算天数差
* 也不好更新答案数组

正确做法：

* 栈里必须存下标

---

### 2. while 写成 if

这里不能只弹一次，因为当前温度可能比栈里前面多个位置都高。

所以必须写：

```python
while stack and temperatures[i] > temperatures[stack[-1]]:
```

不能写成 `if`。

---

### 3. 不理解为什么答案初始化成 0

因为有些位置后面就是没有更高温度，所以默认值就是 `0`。

---

### 4. 不清楚栈为什么是“递减”

递减的是：

**栈中下标对应的温度值**

不是下标本身。

---

## 复杂度分析

* 时间复杂度：`O(n)`
  每个下标最多入栈一次、出栈一次。

* 空间复杂度：`O(n)`
  最坏情况下所有元素都在栈中，例如温度一直递减。

---

## 面试回答模板

这题本质上是求每个位置右边第一个更大元素，所以我会使用单调栈来做。具体来说，我维护一个单调递减栈，栈里存的是还没有找到更高温度的下标。遍历数组时，如果当前温度大于栈顶下标对应的温度，就说明当前这一天是栈顶那一天右边第一个更高温度，于是不断弹栈并计算相差天数。最后栈中剩下的位置说明右边没有更高温度，答案保持 `0` 即可。时间复杂度是 `O(n)`，空间复杂度是 `O(n)`。

## 一句话记忆

**右边找更大，想到单调栈；当前更大时，弹栈并结算。**
